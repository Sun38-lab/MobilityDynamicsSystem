### プロジェクト概要 (Overview)

**PC連動型 1軸姿勢制御（ジンバル）システム**

本プロジェクトは、STM32マイコンによるリアルタイム姿勢制御と、C#で構築したPC側の可視化・チューニングツールを一気通貫で連携させたシステムです。

通常のPID制御（位置制御）では、安価なサーボモーター特有の静止摩擦やギアのガタつきにより、目標値付近で激しい発振（ハンチング）が発生します。本システムでは、PID出力を絶対位置ではなく操作量の増分（スピード）として扱う**速度型PID制御（Velocity-form PID）**モデルを採用しています。これにより、モード切り替え時の出力急変（ショック）を防ぎ、さらに偏差の大きさに応じて動的に制御ロジックを切り替える（ゲインスケジューリング）ことで、産業用機器に迫る安定した着地を実現しています。

また、組み込み領域において「ブラックボックス化しやすい物理現象と制御状態」を、自作のPCツールを用いて波形として可視化することで、理論的かつ効率的なシステム開発・デバッグ環境を構築しました。

### 開発の背景と目的 (Motivation)

業務系ソフトウェア（C#等）の開発経験を活かし、モビリティ・重工分野の組み込みシステム開発へアプローチするためのポートフォリオとして開発しました。
「マイコン内で完結させる」のではなく、「PC側で計測・制御UIを自作し、システム全体を俯瞰して物理的な課題を解決する」という**システムズエンジニアリング**のアプローチを実践しています。

### 💡 コア技術：非線形要素を考慮した状態遷移型ハイブリッド制御

単一の線形制御理論では対応できない安価なアクチュエータの物理特性（静止摩擦、バックラッシュ）を克服するため、偏差に応じた4つのフェーズに分けた状態遷移ベースのハイブリッド制御（簡易ゲインスケジューリング）を実装しています。

**1. 大幅な外乱印加時の非線形制御（バンバン制御）（偏差 ±10度以上）**
外乱等で姿勢が大きく崩れた際は、PIDの計算をキャンセルし、モータを最大速度（PWM変化量 ±3000.0）で一気に目標方向へ引き戻します。これにより、立ち上がり時間（Rise Time）の遅れを最小限に抑えます。

**2. 過渡応答フェーズ（偏差 ±4度〜10度）**
摩擦の影響が少ないこの領域では、強めの比例ゲイン（Kp）を掛けて一気に目標付近まで寄せていきつつ、微分制御（D）でブレーキをかけ始め、オーバーシュートを抑制します。

**3. 不感帯（デッドバンド）の導入とアクティブ制動（不感帯突入時：±2.0度）**
目標付近での着地ショックと、ギアのバックラッシュによる発振を防ぐための最重要フェーズです。
* **ソフトバウンダリの導入:** 物理的なガタつきが生じる±2.0度を「不感帯」として定義し、はみ出した分だけを有効な誤差（active_error）として抽出することで不要な発振をソフトウェア的にマスクします。
* **ノイズの足切りとブレーキ:** ジャイロセンサーの角速度ノイズを足切りしつつ、機体が動いている間（運動エネルギーがある状態）は微分ゲイン（Kd）による制動を優先的に効かせます。

**4. 積分制御の限定適用による定常偏差の解消（アンチワインドアップ）**
不感帯の中で完全に静止したことを検知した状態でのみ、積分ゲイン（Ki）を有効化します。これにより、積分器の過剰な蓄積（ワインドアップ）による暴走を防ぎつつ、微小な静止摩擦をジワジワと突破し、最終的に目標値へ完全に「吸い付く」着地を実現します。

### システムアーキテクチャ

#### 組み込み側 (STM32 / C言語 / FreeRTOS)
* **高精度なタスクスケジューリング:** `osDelayUntil` を用いて、センサー読み取りタスク（優先度：高）を厳密に10ms周期で実行しています。取得したデータはFreeRTOSのメッセージキューを介してUART通信タスク（優先度：低）へ渡され、I/O待ちによる制御周期の崩壊を防いでいます。
* **相補フィルターによる姿勢角推定:** 加速度センサー（長期的な安定性）とジャイロセンサー（短期的な応答性）の利点を組み合わせ、以下の相補フィルター式でノイズの少ないピッチ角・ロール角を算出しています。
  
$$
\theta_{comp} = \alpha (\theta_{comp} + \dot{\theta}_{gyro} \Delta t) + (1-\alpha) \theta_{acc}
$$
* **ハードウェア・フェールセーフ:** I2Cバスのハングアップ対策として、初期化時にSCL/SDAピンをGPIO出力として手動でトグルさせ、STOPコンディションを強制発行してバスロックを物理的に復旧させる堅牢な初期化シーケンスを実装しています。

#### PC側 (C# Windows Forms / ScottPlot)
* **リアルタイムモニタリング:** 受信したシリアルデータから、補正後のピッチ角やジャイロ角速度を50ms間隔でScottPlotを用いてリアルタイムに波形描画します。
* **オンザフライでのパラメータチューニング:** 再コンパイルすることなく、GUI上からUART経由でKp、Ki、Kdの各ゲインや目標角度を稼働中に動的に変更可能です。
* **緊急停止・通信異常検知:** STOPコマンドを送信、あるいはPCとの通信が途絶した場合、STM32側の状態遷移をエラー状態（STATE_ERROR）へ移行させます。即座にタイマーのPWM出力を停止（フリー状態）し、モーターの暴走を物理的に防ぐ安全機能（フェールセーフ）を備えています。

### 開発環境とセットアップ

1. **IDE:** STM32CubeIDE, Visual Studio 2022
2. **ビルドと実行手順:**
    * 本リポジトリをクローンします。
    * `Firmware` フォルダ内のプロジェクトを STM32CubeIDE で開き、ビルド・マイコンへの書き込みを行います。
    * 機体を水平な場所に置き、リセットボタンを押します。約1秒間のジャイロキャリブレーション（オフセット計算）が自動で実行されます。
    * `Dashboard` フォルダのC#ソリューションをビルドし、アプリを立ち上げます。正しいCOMポートを指定するとリアルタイム描画が開始されます。
